# 小米平板桌面 LSPosed 模块

## 一、上滑手势修改

在小米平板的桌面中，应用内底部上滑手势存在两段逻辑：

1. 第一次从底部上滑：显示 Dock 栏。
2. Dock 已显示后再次从底部上滑：理论上应当让当前应用跟随手指上滑缩小，并进入后台、最近任务或返回桌面。

**原始逻辑**

- 第二次上滑时 Dock 仍会优先跟随手指继续上移一段距离，直到 Dock 移动到回弹上限后，当前应用才开始跟手缩小。这会造成应用手势响应滞后。

**修改后的行为**

- 应用内第一次底部上滑仍然正常唤出 Dock。
- Dock 已经显示，或正在显示动画过程中，再次从底部上滑时，当前应用立即开始跟手上滑缩小。
- 不再等待 Dock 跟手移动到回弹上限

## 二、后台动画修改

修改原本进后台的动画，减少原本**劣质感**十足的回弹，改为更舒服的回弹参数。
